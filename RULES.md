# unc-system 운영 규칙

팀챗 메시지 작성 규칙. 봇이 정확하게 집계하려면 아래 형식을 지켜주세요.

---

## 1. 메시지 형식

### 업무계획 (아침)

```
MM.DD 이름 업무계획
• 태스크1
• 태스크2
• 태스크3
```

**예시**
```
04.23 Charco 업무계획
• 듀오링고 영어
• 운동
• 경제신문 읽기
• 기획서 작성
```

### 업무보고 (저녁)

```
MM.DD 이름 업무보고
• 태스크1 O
• 태스크2 X
• 태스크3 세모
```

**예시**
```
04.23 Charco 업무보고
• 듀오링고 영어 O
• 운동 O
• 경제신문 읽기 X
• 기획서 작성 세모
```

---

## 2. 완료 표기 규칙

| 상태 | 표기 | 설명 |
|------|------|------|
| 완료 | `O` 또는 `ㅇ` | 완전히 완료 |
| 미완료 | `X` 또는 `x` | 하지 못함 |
| 부분완료 | `세모` | 일부만 했거나 진행 중 |

- 표기는 태스크 뒤에 붙여씁니다: `• 운동 O`
- 괄호 안 설명 추가는 자유롭게 가능합니다: `• 기획서 작성 세모 (초안까지 완료)`
- **완료/미완료 표기가 없으면 미완료로 처리됩니다**

---

## 3. 날짜별 메시지 분리 규칙

**날짜가 다른 내용은 반드시 별도 메시지로 분리해서 보내주세요.**

❌ 잘못된 예시 (한 메시지에 두 날치)
```
04.01 Charco 업무보고
• 운동 O
• 듀오링고 영어 O

04.02 Charco 업무계획
• 운동
• 경제신문 읽기
```

✅ 올바른 예시 (메시지를 나눠서 전송)
```
[메시지 1]
04.01 Charco 업무보고
• 운동 O
• 듀오링고 영어 O
```
```
[메시지 2]
04.02 Charco 업무계획
• 운동
• 경제신문 읽기
```

한 메시지에 여러 날치 내용이 감지되면 봇이 안내 메시지를 보냅니다.

---

## 4. 코멘트 작성

보고 중 추가 설명이나 코멘트는 자유롭게 작성해도 됩니다.
단, 업무계획에 없던 항목은 집계에서 제외되고 코멘트로 처리됩니다.

**예시**
```
04.23 HYOM 업무보고
• 듀오링고 영어 ㅇ
• 대청소 X

회사 일로 지쳐서 대청소까지 힘들었어요 😅
```

→ `회사 일로 지쳐서 ...` 줄은 코멘트로 처리, 집계 제외

---

## 5. 텍스트 전용 규칙

- 업무계획/보고는 **채팅창에 텍스트로 직접 작성**해주세요.
- 이미지, 스크린샷, 외부 문서 링크(Notion/Google Docs)만 올리면 봇이 인식하지 못합니다.
- 외부 문서 링크는 텍스트 보고 아래에 **참고용으로 추가**하는 건 괜찮습니다.
